## Approach

Keep forward butterfly sums lazy; one 8p->2p reduce for the leg-0 output
In the non-trivial forward radix-4 butterfly, s0=a+c, s2=b+d and s1=a+2p-c
feed only the leg-0 output and the output Plantard multiplies (which tolerate
any non-negative value < 8p). So they no longer need per-term reductions and
stay lazy in [0,4p): s0/s2 are reduced only once, folded into the single
untwiddled output y0 via a two-step 8p->2p reduction (y0 must be [0,2p) for
the next stage). The twiddled outputs y1,y2,y3 (now in [0,6p)/[0,8p)) go
straight through Plantard. Net: one fewer conditional reduction per butterfly.
The trivial-twiddle path is unchanged (its outputs are reduced, not Plantard'd).
