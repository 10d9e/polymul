## Approach

Hoist stage twiddle loads out of the block loop (butterfly-outer iteration)
In both the forward strided stages and the inverse middle DIT stages the
stage twiddle factors depend only on the butterfly column, not on the
block index, yet the loops were block-outer / butterfly-inner, so every
block reloaded the same twiddle vectors. Swapping the loop nesting to
butterfly-outer / block-inner makes the twiddles loop-invariant for the
inner loop, so each twiddle vector is loaded once and reused from
registers across all blocks. The arithmetic is identical; only iteration
order changes (each element is still written exactly once).
