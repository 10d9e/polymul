## Approach

Transform both multiply operands in a single pass. The forward NTT of `a` and of
`b` use identical twiddle factors and identical index arithmetic, so run them in
lockstep: a shared `r4_bfly` butterfly helper is called twice per (block,
position) with the **same** loaded twiddles (`w^e`, `w^{2e}`, `w^{3e}` and their
Shoup constants) and the same index math, instead of loading them once per
transform. Under the weighted cost model a table load costs 2, so halving the
twiddle-load and index traffic across the six forward transforms is a real win.
Outputs are identical; this is purely sharing redundant work.
