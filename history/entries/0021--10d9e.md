## Approach

Fuse the forward NTT's last two radix-4 passes (stages L=8,4 and L=2,1) into a single radix-16 pass over each 16-block: load 16 values, run all four radix-2 DIF stages in registers with lazy sums, store back. Forward is now 3 memory passes (r8, r8, r16) instead of 4. The last stages' twiddles are constant 16th-root powers (independent of block position), so no twiddle table is needed. The only multiplies are on differences bounded by < 8p, so products stay < 8p*p < 2^63 -- the 16p-range values appear only in the final, multiply-free stage, so there is no overflow. Same butterfly network => bit-reversed output unchanged; ~5.6% fewer fuel operators.
