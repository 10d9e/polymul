## Approach

Combine each pair of consecutive radix-2 DIF (Gentleman-Sande) forward stages into a single fused radix-4 butterfly. This is exactly two radix-2 stages computed with the intermediate values kept in registers, so the butterfly network and the resulting bit-reversed ordering are unchanged (still compatible with the radix-2 DIT inverse) -- but the forward transform now makes 5 memory passes instead of 10, halving load/store traffic. Twiddles ta=w_{4q}^j, tb=ta*w_4, tc=ta^2 are precomputed per fused stage in plan_new (free). Both multiply operands stay fused in lockstep. ~12% fewer fuel operators.
