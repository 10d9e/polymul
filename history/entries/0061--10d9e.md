## Approach

Signed-tolerant Plantard (caddv 2^32+2): drop bias adds on difference legs
The Plantard rounding/un-centring constant was 2^32+1, which is exact only
for non-negative inputs, forcing every modular-multiply input that is a
difference `u - v` of two lazy operands to carry a `+kq` bias so it stays
non-negative. Changing the constant to 2^32+2 makes Plantard exact for
SIGNED inputs in (-32q, 40q) as well (verified exhaustively for all three
primes; our intermediates stay within (-8q, 8q)). A Plantard's output is
always unsigned [0,2q) regardless of its input's sign, so any difference
leg that feeds only a Plantard can be passed bare — no bias add.
Applied to every such leg: the forward radix-4 butterfly (r4_lazy_l), both
inverse butterflies (r4_lazy_dit_l and its final variant), the first inverse
sub-stage (dit_l1_in2p), and the Garner CRT (t1 = r1 - v0, t2 = r2 - w). The
sum legs that feed a reduction (red8p/red2p) or the Montgomery pointwise stay
unsigned, as required.
Because a reduced leg's `a` input can now be a signed-negative value, the
scalar correctness-path L::ge is switched to a signed compare to match wasm's
i64x2_ge (they agreed before only because all values were < 2^63); the native
and metered paths remain bit-identical.
Also adds compact contiguous twiddle tables (m16/m64) for the strided inverse
middle stages so each per-column twiddle loads with one L::load instead of two
scattered scalar loads.
