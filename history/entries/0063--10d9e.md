## Approach

Fuse inverse len=16 + len=64 stages into one radix-16 pass
Mirror of the forward radix-16 fusion on the inverse DIT side. The two strided
inverse middle stages (half-blocks 16 then 64) each round-tripped the spectrum
through linear memory; fuse them into a single radix-16 = radix-4(len=16) ∘
radix-4(len=64) pass: gather the 16 elements at stride 16 into a register tile,
run both DIT sub-stages, scatter back — one memory pass instead of two.
As in the forward case, the tile is held in 16 named locals (so it scalarizes
under the strided gather/scatter instead of spilling) and the loop is
column-outer so the twiddles load once and reuse across the four 256-blocks.
The two lanes carry adjacent columns (jj, jj+1); sub-stage A (len=16) shares
twiddle w^{-16*jj}, sub-stage B (len=64) uses w^{-4*(jj+16k)} per group k, both
read as lane pairs from the compact m16/m64 tables.
Bit-identical (same wasm checksums; native gate and all 32 fixtures pass).
