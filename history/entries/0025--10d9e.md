## Approach

Convert both radix-4 transforms to **Harvey-style lazy reduction**. The
division-free Shoup multiply is split into a *lazy* form `shoup_lazy` that
computes `x*c mod p` but leaves the result in `[0, 2p)` — it omits the final
conditional subtraction entirely. This is valid because, with the 2^32 Shoup
constant `c' = floor(c*2^32/p)`, the quotient estimate is off by at most 1 for
*any* input `x < 2^32` (Harvey, "Faster arithmetic for NTTs"); the butterflies
keep all values `< 4p < 2^32`, so inputs are always in range.

Each butterfly now keeps its lattice of values in `[0, 2p)` (intermediate sums
in `[0, 4p)`), reducing with a single `>= 2p` subtract only where needed, and
every twiddle multiply (and the 4th-root multiply) uses `shoup_lazy` with no
conditional subtract. The net effect is roughly one fewer conditional subtract
per modular multiply plus far fewer sum-reductions, versus the strict-`[0,p)`
eager version of entry 0024 — same exact outputs, fewer operations.
