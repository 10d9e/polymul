## Approach

Vectorize the spectral pointwise Montgomery product in the fused boundary. The product is a*b within each packed (a,b) lane vector — a cross-lane horizontal multiply that does not vectorize directly. Instead, deinterleave two adjacent positions with i64x2_shuffle: ta[k] and ta[k+1] become an a-vector (a_k, a_{k+1}) and a b-vector (b_k, b_{k+1}). One vector Montgomery product then yields both spectral products, stored contiguously. The vector mont_mul keeps the same low-32 / multiply-high structure as the scalar one (mask + pinv multiply + reduce), result in [0,2p). Same two-lane L abstraction; native correctness gate runs the identical algorithm and wasm checksums match the scalar baseline on all 32 fixtures.
